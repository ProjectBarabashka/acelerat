Architecture #architecture

TurboTX exposes two API layers. Understanding which to use is essential.

🔓
Layer 1
Public Router API
https://acelerat.vercel.app/api/<endpoint>
9 clean endpoints served by api/router.js via Vercel rewrites. No auth needed. Rate-limited by IP. Use for open integrations, status pages, and wallets.
🔑
Layer 2
Partner API (v1)
https://acelerat.vercel.app/api/v1?method=<name>
Authenticated via API key (ttx_live_...). Same data as Router + TX submission (accelerate). Tiered rate limits.
💡
Router endpoints are also available via v1: /api/health = /api/v1?method=health. They return identical data — use Router unless you need TX submission or higher rate limits.
Router API https://acelerat.vercel.app/api/
Partner v1 https://acelerat.vercel.app/api/v1
Router API — Direct Endpoints #router

All endpoints below are served by api/router.js. Use these clean URLs directly — no authentication required.

Quick Start #quickstart

Get your first API response in under 2 minutes.

1
Check the network (no auth needed)
The Router API requires no key — call it directly from any client.
cURL 
curl "https://acelerat.vercel.app/api/price"
2
Analyze a stuck transaction
Get smart acceleration advice — no key needed for the Router API.
cURL 
curl "https://acelerat.vercel.app/api/price"  # No auth needed!
3
Get a Partner API key (for TX submission)
Contact @turbotx_support on Telegram. Keys look like: ttx_live_...
JavaScript 
const res = await fetch(
  'https://acelerat.vercel.app/api/status?txid=YOUR_TXID'  // Router API — no auth,
  { headers: { 'Authorization': 'Bearer ttx_live_YOUR_KEY' } }
);
const data = await res.json();
console.log(data.status);            // 'mempool' | 'confirmed' | 'not_found'
console.log(data.feeRate);           // e.g. 12 (sat/vB)
console.log(data.accelerationAdvice); // { action, urgency, text }
4
Accelerate a stuck TX
Submit the transaction to 30 broadcast channels covering ~88% of Bitcoin hashrate.
JavaScript 
const res = await fetch('https://acelerat.vercel.app/api/v1?method=accelerate'  // Partner v1 — auth required, {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer ttx_live_YOUR_KEY',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    txid: 'YOUR_64_CHAR_TXID',
    plan: 'premium',  // 'free' | 'premium'
  })
});
const data = await res.json();
console.log(data.ok);             // true
console.log(data.summary);        // { okCount, totalChannels, hashrateReach }
console.log(data.waveStrategy);   // { waves, label } — adaptive 3/5/8 waves
Authentication #auth

All API requests require an API key. Three methods are supported — Bearer header is recommended.

✦ Recommended — Authorization Header
Authorization: Bearer ttx_live_...
X-API-Key Header
X-API-Key: ttx_live_...
Query Parameter (dev/test only)
?apikey=ttx_live_...
⚠ Never expose keys client-side
Keep keys in env vars or a backend proxy
🔓
All endpoints below (/api/health through /api/acceleration) are the Router API — no auth, rate limited by IP. See full response schemas in the Partner API section below.

Error Codes #errors

All errors return a JSON body with ok: false and an error string.

HTTPCode / ReasonResolution
200 ok: false in body TX not found, already confirmed, or method returned empty result
400 Invalid txid / method Ensure txid is exactly 64 hex chars. Check method name against availableMethods in error body.
401 Unauthorized API key missing or invalid — check Authorization: Bearer header
403 Forbidden (bot UA) Request blocked as automated bot. Use a real HTTP client with standard User-Agent.
404 TX not found Transaction not in mempool or blockchain. May have been evicted.
429 Rate limit exceeded Slow down or upgrade tier. See retryAfter in response.
500 Internal server error Upstream Bitcoin node error or service issue. Retry with exponential backoff.
Error Response Body
{
  "ok": false,
  "error": "Rate limit exceeded (per_minute)",
  "limit": 30,
  "retryAfter": 45000,         // ms until limit resets
  "upgradeUrl": "https://acelerat.vercel.app/api-docs#pricing"
}
Unknown Method Error
{
  "ok": false,
  "error": "Unknown method: \"foo\"",
  "availableMethods": [
    "ping", "status", "mempool", "fees", "price",
    "acceleration", "accelerate", "health", "stats", "cpfp", "rbf"
  ],
  "docs": "https://acelerat.vercel.app/api-docs"
}
